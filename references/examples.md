# Examples

## 1. 单需求示例

用户输入：

```text
先别写代码，先帮我把注册流程优化需求拆成可执行步骤。
```

推荐输出方向：

- 判断为单需求
- 明确交付物：注册流程优化方案
- 写清完成标准、错误判断标准、范围边界
- 把任务拆成若干可验证 Step
- 每个 Step 后给复核结论

## 2. 多需求总方案示例

用户输入：

```text
我想优化首页视觉、补登录异常提示、加夜间模式，再顺手整理文档，先帮我排优先级。
```

推荐输出方向：

- 判断为多需求
- 拆成 R1 首页视觉优化、R2 登录异常提示、R3 夜间模式、R4 文档整理
- 标注类型、优先级、依赖、风险
- 输出推荐批次和实施顺序
- 先停在阶段 A 总方案

## 3. 继续拆独立方案示例

用户输入：

```text
根据刚才的总方案继续拆，把 R1 和 R2 分别展开。
```

推荐输出方向：

- 不重新生成总方案
- 直接进入阶段 B
- 将 R1 视为单需求完整展开
- 将 R2 视为单需求完整展开
- 保持与总方案中的目标、边界、优先级一致

## 4. 只做模式判断示例

用户输入：

```text
你先判断一下，这是一件事还是多件事，不要先展开。
```

推荐输出方向：

- 只输出单需求 / 多需求判断
- 给判断依据
- 给拆分后的独立事项清单
- 不展开完整方案

## 5. 只做复核示例

用户输入：

```text
先复核一下这一步对不对，别继续往下做。
```

推荐输出方向：

- 只检查当前步骤的目标是否达成
- 对比执行前后状态
- 给出继续 / 返工 / 暂停建议
- 不提前进入下一步
